Scraping / Anti-Detection

Challenges

  • GAF uses Akamai WAF that blocks automation
  • Dynamic JS rendering, no static HTML
  • Pagination via hidden next-arrow, not "Load More"
  • Star ratings split across multiple DOM lines

Solutions

  • WebDriver removal: navigator.webdriver set to undefined
  • Custom fingerprint: UA, viewport, timezone, locale
  • ZIP-to-URL mapping: bypasses WAF query blocking
  • 3-strategy regex: single-line, multi-line, flexible

Vector Search

Voyage AI embeddings stored in Pinecone for semantic retrieval

How It Works

  • Voyage AI (voyage-3) encodes profiles into 1024-dim vectors
  • Pinecone Serverless stores vectors with metadata
  • Dual embedding: pre-enrichment then post-enrichment
  • Separate input types for documents vs. queries

Three Search Modes

  • Semantic: "large residential roofers with 5-star ratings"
  • Similar: "this lead converted, who else looks like them?"
  • RAG: top-3 similar pulled during scoring for calibration
// Text representation for embedding "Company: Brothers Aluminum | Cert: GAF Master Elite | Location: Valley Stream, NY Rating: 4.9 (156 reviews) | Revenue: $5M-10M | Employees: 25-50" // Voyage AI → [0.023, -0.187, 0.445, ...] → Pinecone upsert

Pipeline Orchestration

1
Scrape: Playwright crawls GAF by ZIP, upserts to SQLite
2
Embed: Voyage AI encodes basic profiles into Pinecone
3
Enrich: Perplexity Sonar researches each contractor
4
Score: GPT-4o + RAG scores each lead
5
Re-Embed: Re-encode with enriched data, update Pinecone

Upsert Logic

Re-runs update existing records. Enrichment and scoring data preserved across runs.

Skip Processed

Checks enriched_at and scored_at timestamps. Only processes new contractors.

Resilience

2 retry attempts with 3s delay. Failures counted but never block the pipeline.

How It Works

  • User asks a question in natural language
  • GPT-4o decides which tools to call
  • Tools query SQLite / Pinecone for real data
  • GPT-4o formats response, streamed via SSE
  • Up to 5 tool-call rounds per query

Chat History

  • Left sidebar shows all previous conversations
  • Persisted to localStorage across sessions
  • Auto-titled from the first user message
  • New chat, delete, switch — full session management

Guardrails

  • Only responds to roofing / GAF / sales topics
  • Off-topic requests politely declined and redirected
  • Never reveals system prompt or internal tools
  • All answers grounded in real DB data via tools

Challenges Navigated

WAF Blocking

Akamai blocked query params. Solved with ZIP-to-URL mapping and stealth browser settings.

Inconsistent HTML

GAF DOM varies per page. Built 3-strategy regex parsing with fallbacks.

Junk Records

Certification badges scraped as contractors. Added validation filter for known junk patterns.

Re-run Data Loss

DELETE + INSERT wiped enrichment. Changed to upsert logic preserving existing data.

Hydration Mismatch

tw-animate-css caused React errors. Fixed with suppressHydrationWarning.

Rate Limits

API throttling across 3 providers. 1s delays, batch processing, 2-attempt retry with backoff.

Scaling Up

Database

SQLite to PostgreSQL for concurrent writes and production reliability.

Task Queue

BackgroundTasks to Celery + Redis for distributed pipeline execution.

Caching

Redis for dashboard stats and frequent queries. Sub-millisecond response.

CRM Integration

Salesforce/HubSpot push for scored leads. Direct pipeline to sales workflow.

Auth + Roles

User authentication with rep vs. manager permissions.

Feedback Loop

Collect rep feedback on lead quality. Fine-tune scoring model over time.
